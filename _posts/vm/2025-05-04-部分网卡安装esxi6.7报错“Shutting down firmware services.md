---
title: "部分网卡安装esxi6.7报错“Shutting down firmware services"
tags: vm
---



```
“Shutting down firmware services…
Using 'simple offset' UEFI RTS mapping policy
Relocating modules and starting up the kernel…”
```

网上查找了相关资料，解决方法如下：

1、在ESXi安装开始前有一个[倒计时](https://so.csdn.net/so/search?q=倒计时&spm=1001.2101.3001.7020)，这时按下 Shift + O，在显示的命令后添加 ignoreHeadless=TRUE，回车， 安装就能正常进行
2、以下的步骤是把这个参数值写入配置， 不会随重启丢失；
3、当ESXi启动后，点击F2， 登录；
4、在System Customization下，进入Troubleshooting Options；
5、启用ESXi Shell，然后一路按ESC返回；(ssh ok)
6、在主界面下，按Alt +F1访问控制台；
7、登录，输入并执行以下命令：

\#进入维护模式

```cobol
vim-cmd /hostsvc/maintenance_mode_enter
```

\#检查主机是否在维护模式

```cobol
vim-cmd /hostsvc/hostsummary | grep inMaintenanceMode
```

\#设置ESXI开机忽略显卡错误

```csharp
esxcfg-advcfg --set-kernel "TRUE" ignoreHeadless
```

\#检查命令

```csharp
esxcfg-advcfg --get-kernel ignoreHeadless
```

\#退出维护模式

```cobol
vim-cmd /hostsvc/maintenance_mode_exit
```

8、输入exit，退出控制台登录；
9、点击 Alt+F2，返回 ESXi 主界面；
10、禁用 ESXi Shell；
11、完成。
